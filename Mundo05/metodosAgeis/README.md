# Métodos Ágeis

Atividade voltada à aplicação de práticas de desenvolvimento ágil utilizando backlog de produto, histórias de usuário e planejamento de sprints.

##

## Sobre

Este projeto simula o planejamento de desenvolvimento de um novo produto bancário utilizando metodologias ágeis.

O cenário envolve a criação de um aplicativo mobile capaz de permitir que turistas estrangeiros realizem pagamentos em moeda local (Real) utilizando saldos em suas moedas de origem.

O processo inclui definição de funcionalidades, organização de backlog, priorização de histórias de usuário e planejamento de sprints.

##

## Objetivo

Aplicar conceitos de **metodologias ágeis** no planejamento de um produto de software, incluindo:

* criação de backlog de produto
* definição de histórias de usuário
* estimativa de esforço (story points)
* definição de prioridades
* planejamento de sprints

##

## Estrutura do Backlog

O backlog do produto foi organizado com os seguintes elementos:

* **Backlog**
  Lista de funcionalidades do produto.

* **Sprint**
  Iterações de desenvolvimento nas quais as histórias serão implementadas.

* **Histórias de Usuário**
  Estruturadas no formato:

  ```
  Eu, como <tipo de usuário>
  desejo <funcionalidade>
  para que eu possa <benefício>
  ```

* **Pontos (Story Points)**
  Estimativa relativa de esforço para implementação.

* **Prioridade**
  Classificação da importância da funcionalidade no produto.
