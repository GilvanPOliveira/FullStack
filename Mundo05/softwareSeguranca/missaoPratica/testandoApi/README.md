# Missão Prática — Refatoração de Segurança em API

Exercício prático focado na identificação e correção de vulnerabilidades em uma aplicação web.

##

## Sobre

Esta atividade simula um cenário real de segurança em software, onde uma aplicação legada apresenta falhas críticas que permitem acesso não autorizado a recursos.

O problema central envolve o uso inadequado de um identificador de sessão (`session-id`), vulnerável a ataques de força bruta, além da ausência de validação de entradas, abrindo espaço para ataques de injeção.

O objetivo é refatorar a aplicação, aplicando boas práticas modernas de segurança para garantir proteção de dados e controle de acesso adequado.

##

## Objetivo

Aplicar práticas de segurança em aplicações web, incluindo:

* substituição de mecanismos inseguros de autenticação
* implementação de autenticação baseada em tokens (JWT)
* proteção de endpoints
* validação de dados de entrada
* mitigação de vulnerabilidades conhecidas

##

## Conceitos Aplicados

* autenticação com JWT
* controle de acesso baseado em perfil (RBAC)
* segurança em APIs REST
* proteção contra ataques de força bruta
* prevenção de SQL Injection

##

## Ferramentas e Tecnologias

* APIs REST
* JWT (JSON Web Token)
* Postman / Insomnia
* validação de requisições
* boas práticas de segurança em backend
