# Microatividade 05 — Tratamento de CRLF Injection

Exercício focado na identificação e correção de vulnerabilidades relacionadas a **CRLF Injection** em aplicações web.

##

## Sobre

Esta atividade aborda uma vulnerabilidade conhecida como **CRLF Injection**, que ocorre quando entradas fornecidas por usuários são utilizadas sem validação adequada na construção de cabeçalhos HTTP.

Esse tipo de falha pode permitir que usuários mal-intencionados manipulem a estrutura das respostas HTTP, provoquem redirecionamentos indevidos ou explorem outras vulnerabilidades da aplicação.

A atividade simula a análise e refatoração de um trecho de código vulnerável para garantir que os parâmetros utilizados em redirecionamentos e cabeçalhos HTTP sejam devidamente tratados.

##

## Objetivo

Aplicar boas práticas de segurança no tratamento de parâmetros utilizados em requisições HTTP, incluindo:

* validação de entradas fornecidas pelo usuário
* prevenção de manipulação de cabeçalhos HTTP
* proteção contra redirecionamentos maliciosos
* fortalecimento da segurança da aplicação

##

## Conceitos Aplicados

* segurança em aplicações web
* prevenção de CRLF Injection
* validação de entradas
* proteção de cabeçalhos HTTP

##

## Ferramentas e Conceitos
